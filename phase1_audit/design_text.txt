Motion specs
Save bar slide-up
transform: translateY(100%) → 0
240 ms · cubic-bezier(0.32, 0.72, 0, 1)
Save bar slide-down (after Save)
transform: 0 → translateY(100%)
200 ms · ease-in · then Petal Rain
History drawer open
translateX(100%) → 0 + overlay fade 0 → 0.08
280 ms · cubic-bezier(0.32, 0.72, 0, 1)
Variables drawer (wide)
same easing; clamped to 760 px or 50% viewport
280 ms
Petal Rain
38 petals · spawn top · wobble + rotation
2800 ms · fade-out last 600 ms
Hover glow cards
translateY(-2px) + double box-shadow sakura
200 ms · cubic-bezier(0.2, 0, 0, 1)
Zen pulse (in flow)
opacity 0.5 ↔ 1, glow 4 ↔ 10 px
2500 ms · ease-in-out infinite
Preview debounce
live preview update on each keystroke
80 ms · trailing
Decisions to remember
prefers-reduced-motion is intentionally NOT respected in v1 (sakura experience prima). Document for a11y review.
Sakura pink reserved to 3 uses: hover/active glow on cards · variable chips {{ }} · feedback animations.
Tag renaming is not supported in v1 — workflow: create new, reassign, delete old.
No “Unsaved changes” copy on the save bar; its presence is the signal.
Skills draft vs saved → no visual differentiation; save bar communicates dirty state.
Render / Raw toggle is disabled while isContentDirty === true.
